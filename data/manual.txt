# 1. Introduction

SmartBank Pro is a digital banking platform designed for retail and corporate customers. The application enables users to perform banking operations such as checking balances, transferring funds, managing cards, paying bills, applying for loans, and monitoring investments.

The purpose of this document is to help users understand the application features and resolve common operational issues while using the banking platform.

## Key Features

* Secure login with multi-factor authentication
* Real-time account balance updates
* Domestic and international fund transfers
* Utility and credit card bill payments
* Loan application and EMI tracking
* Fixed deposit creation and management
* Debit and credit card controls
* Mobile and web banking access
* Instant notifications and alerts

## Supported Platforms

* Web Browser Application
* Android Mobile Application
* iOS Mobile Application
* Tablet Devices

## Browser Compatibility

| --------------- | --------------- |
| Google Chrome   | 110             |

# 3. User Login and Authentication

## Login Procedure

1. Open the SmartBank Pro application.
2. Enter Customer ID or registered mobile number.
3. Enter password.
4. Click on "Login".
5. Enter OTP received on registered mobile number.
6. Access dashboard.

## First-Time Login

Users logging in for the first time must:

* Verify registered mobile number
* Create a secure password
* Set security questions
* Configure transaction PIN

## Password Policy

Passwords must contain:

* Minimum 8 characters
* One uppercase letter
* One lowercase letter
* One numeric digit
* One special character

## Login Error Messages

| Error Message       | Cause                      | Resolution                         |
| ------------------- | -------------------------- | ---------------------------------- |
| Invalid Credentials | Wrong username or password | Re-enter credentials carefully     |
| OTP Expired         | OTP validity exceeded      | Request new OTP                    |
| User Locked         | Multiple failed attempts   | Wait 30 minutes or contact support |
| Session Timeout     | Inactive session           | Login again                        |

## Multi-Factor Authentication

The application supports:

* SMS OTP
* Authenticator App
* Biometric Authentication
* Device-Based Verification

# 15. Troubleshooting Guide

## Application Not Loading

Possible Causes:

* Poor internet connection
* Browser cache issue
* Server downtime

Resolution:

* Refresh browser
* Clear cache
* Restart application
* Check internet connection

## OTP Not Received

Possible Causes:

* Network issue
* DND enabled
* Incorrect mobile number

Resolution:

* Wait 2 minutes
* Request new OTP
* Verify registered number
* Contact telecom provider

## Transaction Stuck in Pending

Resolution Steps:

1. Wait for 30 minutes.
2. Check transaction history.
3. Avoid duplicate transaction.
4. Contact support if unresolved.

## Mobile App Crashing

Resolution:

* Update app
* Clear app cache
* Reinstall application
* Restart device

# 16. Frequently Asked Questions

## Q1. How can I reset my password?

Use the "Forgot Password" option on the login screen and complete OTP verification.

## Q2. What should I do if money is debited but transfer failed?

Wait for 30 minutes. If amount is not reversed automatically, contact customer support.

## Q3. How do I activate international card usage?

Navigate to Card Services → Card Controls → Enable International Usage.

## Q4. Can I increase transfer limits?

Yes. Users can request limit enhancement through profile settings or branch verification.

## Q5. How do I update mobile number?

Go to Profile Settings → Contact Information → Update Mobile Number.

## Q6. What happens if I enter wrong password multiple times?

The account may be temporarily locked for security reasons.

## Q7. How can I download account statements?

Go to Accounts → Download Statement.

## Q8. Is biometric login secure?

Yes. Biometric data is securely encrypted and stored on the device.

# Appendix A – Sample User Scenarios

## Scenario 1 – Successful Fund Transfer

A user logs into the application, adds a beneficiary, completes OTP verification, and successfully transfers INR 10,000 through IMPS.

Expected Result:

* Transaction status should display as "Success".
* SMS and email confirmation should be sent.
* Account balance should update immediately.

## Scenario 2 – Failed Bill Payment

A user attempts to pay an electricity bill with insufficient balance.

Expected Result:

* Payment should fail.
* Error message should display "Insufficient Balance".
* No amount should be debited.

## Scenario 3 – Password Reset

A user forgets password and initiates password reset.

Expected Result:

* OTP verification should complete.
* User should set new password.
* Old password should become invalid.
